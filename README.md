# 运营商客户行为大数据分析系统

**Telecom Customer Behavior Analytics Platform**

> 一个端到端的大数据分析系统，帮助运营商从海量用户数据中识别高价值客户、预警流失风险、驱动精准营销决策。

---

## 这个项目做了什么

运营商每天产生数亿条用户通话、充值、上网记录，但这些原始数据无法直接用于业务决策。本项目搭建了一套完整的数据处理流水线，将 **10 万用户、226 项指标、4 个月** 的原始数据，转化为可直接支撑运营决策的分析报表和实时告警。

**一句话概括：从原始 Excel 表格，到老板能看懂的 dashboard，全流程自动化。**

---

## 能回答哪些业务问题

| 业务场景 | 系统给出的答案 |
|----------|---------------|
| 哪些地区的业绩最好/最差？ | 按 ARPU、活跃度、数据用量三维综合排名，每月自动更新 |
| 哪些客户可能要流失了？ | 自动识别"用量环比骤降 50%+"的客户，分级告警（CRITICAL / WARNING） |
| 哪些沉默客户值得优先挽回？ | 标记"上月高消费但本月零使用"的客户，按流失风险排序，附带建议措施 |
| 各地区有多少高价值客户？ | 按 ARPU≥500、通话>500min、流量>1GB 等 20+ 维度打标签，按月汇总 |

---

## 数据从哪里来

使用 **Kaggle 真实印度运营商预付费客户数据集**（非模拟数据），包含 99,999 名客户在 2014 年 6-9 月的 226 项行为指标，涵盖通话时长、充值金额、流量使用、套餐订阅、漫游情况等。数据规模和数据维度都与真实运营商场景一致。

## 技术栈与工具

| 分类 | 技术 | 在项目中承担的角色 |
|------|------|-------------------|
| 批量数据处理 | **PySpark** (Python) | 核心 ETL 引擎，处理 10 万×226 列数据，完成数仓建模 |
| 数据仓库 | **Hive** (SQL) | 四层数仓（ODS→DWD→DWS→ADS）存储，Parquet 列存 + Snappy 压缩 |
| 实时消息队列 | **Kafka** | 模拟运营商 BSS 计费系统实时推送用户行为事件 |
| 实时流计算 | **Flink** | 5 分钟滚动窗口聚合，实时检测异常行为和流失风险 |
| 数据可视化 | **Plotly** (Python) | 生成交互式 HTML 看板，四合一图表（趋势/对比/分布/排名） |
| 基础架构 | **HDFS / Hadoop** | 分布式文件存储和资源调度 |
| 编程语言 | **Python / SQL** | 数据处理和分析 |

## KPI 指标体系

本项目定义了一套 **20+ 维度**的客户评价体系，对标运营商真实业务场景：

| 指标类型 | 具体指标 | 业务含义 |
|----------|---------|---------|
| **价值指标** | ARPU（每用户平均收入） | 客户值多少钱，≥500 为高价值 |
| **活跃指标** | 月通话分钟、月流量用量 | 客户有多活跃 |
| **行为标签** | 重度通话 / 重度数据 / 漫游用户 / 2G/3G 用户 | 客户画像维度 |
| **风险指标** | 用量环比下降率、沉默状态 | 客户是否要流失 |
| **地区指标** | 活跃率、不活跃率、人均 ARPU、人均用量 | 各地区健康度对比 |

## 快速运行

```bash
# 1. 下载数据（需要 Kaggle API token）
python download_data.py

# 2. 运行批量 ETL 管道（核心流程，无需 Hadoop 集群，本地即可运行）
pip install pyspark pandas
python spark_batch_etl.py

# 3. 运行实时处理演示（本地模拟，无需 Kafka/Flink 环境）
pip install kafka-python
python kafka_producer.py    # 终端 1：模拟数据上报
python flink_processor.py   # 终端 2：实时流失检测

# 4. 生成可视化看板
pip install plotly
python visualization.py     # 打开 output/telecom_dashboard.html
```
